Home Page and Navigation

The home page is located at https://timoun-production.appspot.com. On this page, you will see a welcome message and a brief overview of Timoun.

The Navigation bar has several links, each which lead to a separate page.

About

The about page is meant to give the public more information about Timoun.

Search

This page will allow for users to search for services in the Timoun database. It will also explain how best to search the database. How to use filters, keywords etc to narrow down their search.

This page will explain a little bit about IBESR(Institut du Bien-Etre Social et de recherches), and give contact information for that organization.

Login/Logout

Users and Login/Logout

Add a User

Users are added by an admin.

An Admin will click Users on the navigation bar. After that, the admin will click the Add User button.

After the admin fills out the form, the user will get a confirmation email, which will explain how they can set their password.

View and Delete Users

An Admin can click Users on the navigation bar to see all users. On that page, they can see basic user information, and can click on the email to edit or delete that user.

Forgot Password

If a user forgets their password, they can click Login and click Forgot Password. Clicking either of these links will log you out. So you'll have to log back in if you use them.

From here, a user will enter their email, and receive and email link explaining how they can reset their password safely.

Admin Pages

What can you do as an Admin?

The Admin user has several User Interface changes to take note of. When on a public page, an Admin user has an extra link at the top that says "Admin". To enter the Admin console, click on that link.

Once there, the Admin has an Admin Nav on the left of the screen. This will allow the admin to explore the database, and track changes. There are options to view the Dashboard of all recent changes, as well as Users, Organizations, Programs, Services, Records, and this Manual.

Dashboard and Audits

View Audits on the Dashboard

Timoun keeps track of all changes to the database.

An admin can see all changes by clicking Dashboard on the navigation panel.

The Dashboard displays a table tracking all changes to the system. For instance, if you add a User, or edit a Record, this table will show you who the change was initiated by, what the resource affected was, the security clearance, the resource, the action performed ("Create User", for instance), and the Time. It will also allow you to click "View" to see more detailed information.

This more detailed information will bring you to a separate page, which will show you all changes to that particular resource.

Admin Organizations Panel

View, add, and edit organizations

On the Admin panel, click "Organizations". This will show you a list of Organizations in the system. When you click on it, it will bring you to a Record that lists that organization, and all of it's programs and services.

To edit, click "edit" and proceed to make changes that are present on the form.

To add a new organization, click "Add Organization", and fill out the complete form for that record. This new Record will not be connected with iformbuilder in any way.

To edit, click "edit" and proceed to make changes that are present on the form.

To add a new program, click "Add Program", and fill out the complete form for that record. This new Record will not be connected with iformbuilder in any way.

How Records Interact With Organizations

Each Organization is connected to a record

There is nothing on the database simply called "Organization", there are only Records. When you are on the Organization index page, what you are seeing is a list of the records, shown by the associated organization name. This allows an admin to interact with everything associated with a record, not just the fields that pertain to it's organization.

How Records Interact With Services

Each service can have a list of records

There is nothing on the database simply called "Service", there are only Records. When you are on the Service index page, what you are seeing is a list of all of the Services derived from records in the database. When you click a Service, you will be presented a list of records that provide that service. From there you can click to view or edit that individual service.

How Records Interact With Programs

Each program can have a list of records

There is nothing on the database simply called "Service", there are only Records. When you are on the Service index page, what you are seeing is a list of all of the Services derived from records in the database. When you click a Service, you will be presented a list of records that provide that program. From there you can click to view or edit that individual program.

User Profile

Viewing and editing user information

Any user, whether an Admin or not, can click on their own user profile. This will bring them to a pre-populated form that shows them was information we have for them on the system. To edit, make changes to the form, and hit submit. The changes will be stored on the database.
